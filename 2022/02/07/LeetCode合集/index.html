1. 两数之和

暴力枚举方法很简单：两重循环枚举下标 i,j，然后判断 $nums[i]+nums[j]$ 是否等于 target。
时间复杂度：由于有两重循环，所以复杂度是 $O(n^2)$

使用哈希表，循环一遍nums数组，每次枚举一个数时做两件事

  1. 判断 $target−nums[i]$ 是否在哈希表中；
  2. 将 $nums[i]$ 插入哈希表中；

时间复杂度：由于只扫描一遍，且哈希表的插入和查询操作的复杂度是 $O(1)$，所以总时间复杂度是 $O(n)$.

4. 寻找两个正序数组的中位数

原问题难以直接递归求解，所以我们先考虑这样一个问题：

在两个有序数组中，找出第k小数。

如果该问题可以解决，那么第 $(n+m)/2$ 小数就是我们要求的中位数.

先从简单情况入手，假设 $m,n≥k/2$，我们先从 A 和 B 中各取前 $k/2$ 个元素：

  1. 如果 $A[k/2−1]>B[k/2−1]$，则说明 A 中取的元素过多，B 中取的元素过少；因此 B 中的前 $k/2$ 个元素一定都小于等于第 k 小数，所以我们可以先取出这些数，将问题归约成在剩下的数中找第 $k−⌊k/2⌋$ 小数.

  2. 如果 $A[k/2−1]≤B[k/2−1]$，同理可说明 A 中的前 $k/2$ 个元素一定都小于等于第 k 小数，类似可将问题的规模减少一半.

现在考虑边界情况，如果 $m<k/2$，则我们从 A 中取m个元素，从B 中取 $k/2$ 个元素（由于 $k=(n+m)/2$，因此 m,n 不可能同时小于 $k/2$.）：

  1. 如果 $A[m−1]>B[k/2−1]$，则 B 中的前 $k/2$ 个元素一定都小于等于第 k 小数，我们可以将问题归约成在剩下的数中找第 $k−⌊k/2⌋$ 小数.

  2. 如果 $A[m−1]≤B[k/2−1]$，则 A 中的所有元素一定都小于等于第 k 小数，因此第k小数是 $B[k−m−1]$.

时间复杂度分析：$k=(m+n)/2$，且每次递归 k的规模都减少一半，因此时间复杂度是 $O(log(m+n))$.

10. 正则表达式匹配

状态表示

设状态 $f(i,j)$ 表示字符串 s 的前 $i$ 个字符和字符串 p 的前 $j$ 个字符能否匹配。

这里假设 s 和 p 的下标均从 1 开始。初始时，$f(0,0)=true$。

状态转移

$f(i,j)=f(i,j)$ or $f(i−1,j−1)$，当 $i>0$ 且 s(i) == p(j) || p(j) == '.'。

当 p(j) == '*' 时，若 $j>=2$，$f(i,j)$ 可以从 $f(i,j−2)$ 转移，表示丢弃这一次的 '*' 和它之前的那个字符；

若 $i>0$ 且 s(i) == p(j - 1)，表示这个字符可以利用这个'*'，则可以从$f(i−1,j)$ 转移，表示利用 '*'。

初始状态 $f(0,0)=true$；循环枚举 $i$ 从 0 到 $n$；$j$ 从 1 到 $m$。

因为 $f(0,j)$ 有可能是有意义的，需要被转移更新。

最终答案为 $f(n,m)$。

$matrix[i][j] \ = \ 0$，$f[i][j] \ = \ 0$

$matrix[i][j] \ = \ 1$，$f[i][j] = min(f[i][j - 1],\ f[i - 1][j],\ f[i - 1][j - 1]) + 1$

223. 矩形面积

计算两个矩形相交部分的面积可以通过两个矩形长和宽分别投影在 $x$ 轴和 $y$ 轴部分的长度来计算

计算 $x$ 轴部分投影相交的长度：$ x = max(0, min(ax_2, bx_2) - max(ax_1, bx_1))$

计算 $y$ 轴部分投影相交的长度：$ y = max(0, min(ay_2, by_2) - max(ay_1, by_1))$

相交部分的矩形面积大小：$x * y$

两个矩形覆盖的总面积为：$area = (ax_2 - ax_1) * (ay_2 - ay_1) - x * y + (bx_2 - bx_1) * (by_2 - by_1)$

236. 二叉树的最近公共祖先

dfs(root, p, q)：返回从root根结点往下找是否找到p点和q点的状态码state

state表示找到p点和q点的状态码:

  1. 11表示p点和q点都能找到
  2. 10表示只能找到p点
  3. 01表示只能找到q点
  4. 00表示两个点都找不到

当前点root找到p点和q点的状态主要依赖如下几种情况：

  1. dfs(root -> left, p, q)
  2. dfs(root -> right, p, q)
  3. 当前点为p点，root == p
  4. 当前点为q点，root == q

找到最早能同时找到p点和q点的结点，即最早出现state = 3的结点

使用两个哈希表分别记录父节点，节点深度，通过不断向上追溯父节点，找到最近公共祖先

  1. 使用深度优先搜索遍历一遍整颗树，记录下对应的父节点和深度
  2. 先把深度较深的节点追溯到和另一个节点的深度一致
  3. 两个节点同时向上追溯，直到两个节点相同为止

309. 最佳买卖股票时机含冷冻期

(动态规划) $O(n)$

状态表示

  1. $f(i)$ 表示第 i 天，当前结束不持有股票且当前没有发生卖出交易的最大收益；

  2. $g(i)$ 表示第 i 天结束后不持有股票，且当前刚刚卖出股票的最大收益；

  3. $h(i)$ 表示当前持有股票的最大收益。

状态转移

$f(i)=max(f(i−1),g(i−1))$，表示构成第 i 天不持有股票且当天无交易有两种方式

  1. 一种是前一天也不持有且前一天没有卖出交易
  2. 另一种是前一天持有且前一天刚刚卖出股票；

二者取最大值。

$g(i)=h(i−1)+prices[i]$，表示构成第 i 天不持有股票且当天有交易仅有一种方式，即当天卖出前一天持有的股票。

$h(i)=max(h(i−1),f(i−1)−prices[i])$，表示构成第 i 天持有股票有两种方式：

  1. 一种是前一天持有
  2. 另一种是前一天不持有且前一天无交易，但这一天刚刚买入。

最终答案为 $max(f(n−1),g(n−1))$，即最后一天不持有股票的两种情况。

343. 整数拆分

首先把一个正整数 NN 拆分成若干正整数只有有限种拆法，所以存在最大乘积。
假设 $N=n_1+n_2+…+n_k$,并且 $n_1×n_2×…×n_k$ 是最大乘积。

  1. 显然1不会出现在其中；
  2. 如果对于某 $i$ 有 $n_i≥5$，那么把 $n_i$ 拆分成 $3+(n_i−3)$，我们有 $3(n_i−3)=3n_i−9>n_i$；
  3. 如果 $n_i=4$，拆成 $2+2$ 乘积不变，所以不妨假设没有4；
  4. 如果有三个以上的2，那么 $3×3>2×2×2$，所以替换成3乘积更大；

综上，选用尽量多的3，直到剩下2或者4时，用2。

406. 根据身高重建队列

思路：

  1. 身高高的人只会看到比他高的人，所以当身高高的人固定好了位置，前面插入多少个矮的人都不会破坏高的人的条件限制。

486. 预测赢家

从最简单的问题开始考虑，假设只有一个数字，则只能玩家 A 选择这个数字。

接着，问题的规模开始扩大，扩大后，两个玩家会有两种决策，一种是选择数组头部，一种是选择数组尾部，而这两种情况下的子问题都可以提前计算出。

状态表示

$f(i, j)$表示闭区间$[i, j]$下，玩家A所能获取的最大分数

初始状态：如果最后一次取数是玩家A，则$f(i,i)=nums[i]$，否则$f(i,i)=0$

状态转移

当玩家A取数字时，从头部或者尾部取数字，取二者最优：$f(i, j) = max(f(i + 1, j) + nums[i], f(i,j - 1) + nums[j])$

当玩家B取数字时，B会采用最优解，此时会导致A得分减少：$f(i,j) = min(f(i + 1, j), f(i, j -1))$

最后玩家 A 能获得的最大得分就是 $f(0,n−1)$。

状态表示

设 $f(i,j)$表示闭区间 $[i, j]$ 下，先手玩家与后手玩家的得分差距的最大值。

初始时，$f(i,i)=nums[i]$

状态转移

因为此时是先手玩家，被转移的区间就是后手玩家的得分，所以需要 取相反数 然后转移。

$f(i,j)=max(nums[i]−f(i+1,j),nums[j]−f(i,j−1))$

最后我们只需要判断 $f(i,j)$ 是否大于等于 0。

543. 二叉树的直径

思路

  1. dfs(root)返回的是从该节点往下走的最大值

  2. ans记录最大路径和

  3. 每一层DFS用left,right两个变量记录当前节点左右两个儿子的最大的单向路径和

    1. $left = dfs(u.left)$
    2. $right = dfs(u.right)$

  4. 更新ans,$ans = max(ans, left + right)$

    **注意：**两结点之间的路径长度是以它们之间边的数目表示。

  5. 每一层DFS返回以当前节点为祖宗节点的单向路径的最大和，即$max(left, right) + 1$

  2. 遍历前缀和数组，用 unordered_map 哈希表记录每个前缀和出现的次数。特别地，初始时前缀和为 0 需要被额外记录一次。

611. 有效三角形的个数

  1. 将整个数组从小到大排序。
  2. 在排序后的数组上，从前向后枚举三角形的两个较短的边 nums[i] 和 nums[j]，较短的边不能为 0。
  3. 在确定了最长的边 nums[i] 后，令 j = i - 1，然后再枚举次短的边 nums[k] 后，不断向后更新 k，使得nums[k] + nums[j] <= nums[i]。由于 nums[j] 单调递增，故 也会单调递增。最终对于每次枚举的 nums[j]，累计的答案个数就是 k - j - 1。

781. 森林中的兔子

哈希表 + 数学

用哈希表统计每种数字出现的次数。

不同数字之间的颜色一定是不同的，且每种颜色的兔子一定是 x + 1 的整数倍。

设某个数字 x 的出现次数为 num

如果 num 是 x + 1 的整数倍，则恰好需要有$num / (x + 1)$ 种颜色，且这 num 只兔子都回答了。

否则，需要 $num / (x + 1) + 1$ 种颜色，共有 $(num / (x + 1) + 1) * (x + 1)$ 只兔子回答了 x。
